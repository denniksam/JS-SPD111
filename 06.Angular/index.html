Основи Angular

Angular - JS фреймворк. Існує два види фреймворків: Angular.JS (або Angular-1) та Angular-CLI

Концепція Angular полягає у компонентній моделі, схожій на ООП. Задачі розбиваються на певні групи, для яких створюються компоненти, що є локалізованими утвореннями (мають локальні дані, змінні тощо - віджети) і при цьому взаємодіють з глобальним контекстом. А також реалізація патерну MVVM, який дозволяє зв'язування (та спостереження) з автоматичним оновленням представлення (View) при зміні значень Моделі, яка знаходиться під спостереженням.

Angular впроваджується у HTML та ставить під автоматичне спостереження блок, позначений атрибутом ng-app. Частіше за все, цим атрибутом позначають увесь документ <html ng-app>. Усередині блоку опрацьовуються вирази, для створення виразу використовуються подвійні фігурні дужки { { 3 + 4 } } = {{ 3 + 4 }}

Зв'язування даних відбувається через моделі. Модель може бути створена як у HTML (атрибутом ng-model), так і у коді. Доступ до даних моделі здійснюється через вираз з її іменем: введено - {{username}}

Контролери

Контролери - спосіб утворення та ініціалізації scope - локальної області (лексикографічного околу) застосунку. Див. скрипт та коментарі у ньому. Окіл по-перше дозволяє створити змінні, функції, які доступні всім дочірнім елементам контролера. По-друге, надає програмний доступ до їх значень. По-третє, автоматично ставить на спостереження ці змінні (вважає їх моделями).

data1 = {{data1}}

{{data2}}

Поза контролером data1 = {{data1}}
data1 = {{data1}}

Директиви

В Angular директиви - розширення ідеї контролера зі спрощеним впровадженням у DOM, а також з можливістю включення до свого складу розмітки. Директиви включаються одним з наступних способів

  • restrict: "A" - через атрибут довільного тега
  • restrict: "С" - через ім'я класу
  • restrict: "M" - через коментар
  • restrict: "E" - через ім'я тегу (самий поширений спосіб)
  • restrict: "ACE" - комбінований (довільний набір літер)

Компоненти

Компоненти - це еволюція директив, певна "обгортка" над ними. З найбільш поширеними значеннями за замовчанням: впровадження у теги, ізоляція околу і т.п.
